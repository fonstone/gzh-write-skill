# 文章原型写法指南

## 为什么需要文章原型

框架（frameworks.md）定义了**文章的结构骨架**——每一段写什么、占多少字。原型（archetypes）定义了**写作的叙述立场**——你和读者之间是什么关系，用什么姿态来讲。

同一个框架（比如热点解读型）可以是产品体验型原型（"我亲自用了"），也可以是现象解读型原型（"你发现了吗"）。**原型决定了文章的底层叙事基因**。

---

## 原型一：调查实验型

### 核心叙事基因

**"我替你去做了这件事。"** 亲自下场，边做边报道发现。最能建立信任感和叙事张力。

### 适用场景

- 测评付费产品/服务（"9.9元的XX到底值不值"）
- 验证某个说法/现象（"AI真的能写代码吗？我试了"）
- 揭露行业猫腻/信息差（"我花了500找人上门装XX，结果…"）

### 写法重心

| 要素 | 要求 | 示例 |
|------|------|------|
| **起点** | 读者能代入的具体好奇/困境，非抽象命题 | "我就想看看9.9的XX到底是什么" 而不是"AI工具市场存在严重的价格混乱" |
| **过程** | 每一步都带真实反应——不是干巴巴的步骤 | "于是我就下了单，付了款。然后对方给我发的东西，我差点以为我进平行宇宙了" |
| **发现** | 层层递进，不是一次性抛出结论 | 先展示普通结果 → 再展示怪现象 → 最后揭露本质 |
| **亲自下场** | 读者能感受到你真的做了 | 具体到付款截图、时间、对话记录 |

### 结构节奏

```
起点（具体好奇/困境）
  ↓ 第一步尝试（普通反应）
  ↓ 意外发现（"？？？"时刻）
  ↓ 深入调查（层层跑）
  ↓ 核心洞察（原来如此）
  ↓ 升华（看到更大的东西）
```

### 检查清单

- [ ] 起点是具体可代入的困境/好奇吗？
- [ ] 过程有"意外发现"的层层递进吗？
- [ ] 每一步都有你的真实反应（情绪/吐槽/惊讶）？
- [ ] 读者看完觉得自己"也跟着经历了一遍"？

## 原型二：产品体验型

### 核心叙事基因

**"跟我一起玩。"** 拿到产品实际使用，带读者一起体验。不是评测报告，是"来，我带你看看这个有意思的东西"。

### 适用场景

- 新产品上手体验（App/硬件/平台）
- 新功能深度测试
- 工具玩法展示

### 写法重心

| 要素 | 要求 | 示例 |
|------|------|------|
| **场景驱动** | 每个功能在具体场景中展示 | "当你要批量处理100张图的时候…" 而不是"该产品支持批量处理功能" |
| **真实感受** | 你的真实使用体验，好坏都说 | "这个功能一开始我没搞懂，试了三次才找到入口" |
| **自然对比** | 跟同类产品对比，但像聊天一样顺带提 | "跟XX比，这个设计更…不过也牺牲了…" |
| **"玩"的姿态** | 不是测评，是探索 | "我试了一个骚操作——你看它能不能识别出这个" |

### 结构节奏

```
"我拿到了XX"（第一印象/开箱）
  ↓ 逐个功能体验（每个带实际场景和吐槽）
  ↓ 创意玩法（"我还试了个更骚的"）
  ↓ 深入对比（跟谁比、差在哪、好在哪）
  ↓ 总结谁适合用（给出明确判断）
```

### 检查清单

- [ ] 每个功能都有具体使用场景支撑？
- [ ] 有你的真实感受和吐槽？
- [ ] 有自然的产品对比（不是硬对比）？
- [ ] 看完读者知道"我该不该用"？

## 原型三：现象解读型

### 核心叙事基因

**"你注意到了吗？背后是这样。"** 观察到一个有趣现象，以好奇心驱动层层深入分析，最后连到大画面。

### 适用场景

- 网络文化现象解读（刷屏事件/网络热梗）
- 技术现象分析（"为什么AI看不到爱心"）
- 社会现象洞察

### 写法重心

| 要素 | 要求 | 示例 |
|------|------|------|
| **好奇心驱动** | 从"这有意思"出发，不是"这很重要" | "前两天刷到一张图，很有意思" 而不是"这一现象反映了…" |
| **层层剥开** | 现象→表面解释→更深追问→核心洞察 | 每一步都是"等一下，那为什么…" |
| **知识随手掏** | 学科知识像聊天中想起来的 | "突然想起了以前做交互设计时，有一个刻在血液里的心理学…" |
| **哲学升维** | 最后连到一个更大的文化/哲学参照 | 从AI看不到爱心 → 格式塔心理学 → "我们活在流中，AI活在帧中" |

### 结构节奏

```
现象（具体、有画面、让人"？？？"）
  ↓ 第一层解释（大多数人会怎么想）
  ↓ 追问（"但等一下——"）
  ↓ 深入研究发现（引用论文/数据/专家）
  ↓ 突然的联想（知识随手掏）
  ↓ 哲学/文化升维（"聊着聊着想到了……"）
```

### 检查清单

- [ ] 现象本身足够有趣/反常识吗？
- [ ] 有层层剥开的追问推进吗（不是平铺叙述）？
- [ ] 知识出现时像"聊着聊着想起来的"吗？
- [ ] 升维自然、有"原来如此"感？

## 原型四：工具分享型

### 核心叙事基因

**"我发现了一个好东西。"** 分享一个实用工具/Prompt/方法，但必须用个人故事包裹，不能直接"推荐一个好用的xxx"。

### 适用场景

- 分享Prompt模板
- 推荐效率工具
- 分享某个好用的小技巧

### 写法重心

| 要素 | 要求 | 示例 |
|------|------|------|
| **个人故事铺垫** | 从你的一个真实困境/焦虑出发 | "最近跟朋友聊到了天赋这个词，我突然发现…" |
| **自然引出工具** | 工具不是硬塞的，是解决问题的答案 | "后来我找到一个东西，它叫…" |
| **你自己的使用过程** | 你用了、测试了、拿结果了 | "我自己先试了一下，输入的是…输出的结果是…" |
| **效果展示** | 让人"卧槽"的真实结果 | 具体截图、输出对比、量化数据 |

### 结构节奏

```
你的真实困境/焦虑
  ↓ 尝试了各种方法（都失败了）
  ↓ 偶然发现/朋友推荐了这个工具
  ↓ 自己的测试过程（带截图和数据）
  ↓ 结果展示（让人"卧槽"）
  ↓ 使用建议和注意事项
```

### 检查清单

- [ ] 有个人故事铺垫（不是直接推荐）？
- [ ] 工具出现是因为"它解决了我的问题"？
- [ ] 有具体的测试过程/使用场景？
- [ ] 效果展示有截图/数据/对比？

## 原型五：方法论分享型

### 核心叙事基因

**"我把压箱底的东西掏给你了。"** 把自己长期积累的经验和方法论系统性地分享出来。每一节读完，读者手里都有一个可以今天执行的动作。

### 适用场景

- "我的XX方法论"
- "做XX的X条心得"
- "如何学会XX"

### 写法重心

| 要素 | 要求 |
|------|------|
| **每节可执行** | 每节读完后，读者能做一件事。如果某节读完没有行动输出，那节就是多余的 |
| **结构模板** | 观点 → 案例/理论支撑 → 具体怎么做 → **坦诚说明学习曲线和失败点** |
| **谦逊铺垫** | 开头必须卸掉"我来教你"的傲慢感（"我也不知道行不行""不成熟的经验"）|
| **诚实说代价** | 必须说清楚初期的笨拙期（"一开始可能会花更多时间"），不只画成功画面 |
| **结尾回扣升华** | 回扣所有行动点并升华 |

### 常见格式

**序号法**：1. 2. 3. 这是唯一一种可以用编号的文章类型。每条都是独立的心得/方法。

**注意事项**：
- 每一条都必须有"观点 + 案例/数据 + 行动建议"三步
- 每一条都要说清楚"这条我踩过的坑"
- 每一条都要坦诚说明适用边界（"这个方法不适合X类型的人"）

### 检查清单

- [ ] 每节读完读者能立刻动手做一件事？
- [ ] 明确说明了学习曲线和初期可能的失败？
- [ ] 开头/结尾有谦逊铺垫卸掉傲慢感？
- [ ] 知识不堆砌——每条都有案例/故事支撑？
- [ ] 形式是编号条目（1. 2. 3.）而不是markdown小标题？

## 原型快速对照表

| 原型 | 一句话基因 | 读者代入的身份 | 最适合的框架 |
|------|-----------|---------------|-------------|
| 调查实验型 | "我替你去做了" | 跟屁虫、同行者 | 故事型/热点解读型 |
| 产品体验型 | "跟我一起玩" | 体验伙伴、试用搭子 | 对比型/清单型 |
| 现象解读型 | "你发现了吗" | 好奇的探索者 | 热点解读型/纯观点型 |
| 工具分享型 | "我发现了好东西" | 得益的分享者 | 清单型/痛点型 |
| 方法论型 | "我掏家底给你" | 向学的晚辈 | 清单型/复盘型 |

## 原型选择决策树

```
文章主要想传递什么？
  ├─ "我替你试过了" → 调查实验型
  ├─ "来体验这个好玩的" → 产品体验型
  ├─ "你注意到这个现象了吗" → 现象解读型
  ├─ "我发现了这个好用的" → 工具分享型
  └─ "我积累了一套方法" → 方法论型
```
